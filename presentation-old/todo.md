# План слайдов презентации (Presentation TODO)

## 1. Титульный слайд
- [ ] Тема: "Оптимизация холодного старта и масштабирования моделей машинного обучения в бессерверном инференсе для обеспечения высокой доступности".
- [ ] Автор, Руководитель.

## 2. Актуальность темы
- [ ] Рост размеров ML-моделей (от мегабайт до десятков гигабайт).
- [ ] Переход к Serverless архитектуре в Kubernetes.
- [ ] Проблема медленного скачивания весов (холодный старт), влияющая на доступность сервисов и SLA.

## 3. Цель и задачи
- [ ] Цель: Значительно сократить время холодного старта ML-моделей в распределенных Serverless-системах.
- [ ] Задачи:
  - [ ] Проанализировать существующие решения (Alluxio, JuiceFS).
  - [ ] Разработать механизм синхронизации скачивания.
  - [ ] Спроектировать и внедрить децентрализованную P2P систему кэширования.

## 4. Постановка бизнес-задачи
- [ ] Как было: `storage-initializer` качает веса в `emptyDir`, затем запускается `runtime-container`. Медленно, дублирование трафика, нет переиспользования данных.
- [ ] Анализ рынка: Alluxio и JuiceFS слишком дороги, сложны в эксплуатации и плохо подходят для Serverless.
- [ ] Потребность бизнеса: дешевое, быстрое и отказоустойчивое решение для масштабирования инференса.

## 5. Требования к архитектуре
- [ ] Минимизация сетевого трафика во внешнюю сеть (HuggingFace/Ollama).
- [ ] Отсутствие единой точки отказа (SPOF).
- [ ] Прозрачность для ML-движка (например, `vllm` не должен знать о сети).
- [ ] Высокая скорость локального чтения (скорость диска/памяти).

## 6. Архитектура
- [ ] **Эволюция:** от NFS к P2P.
  - [ ] Проблемы NFS: единая точка отказа, `flock over nfs`, Leader Election.
- [ ] **Новая P2P архитектура (схема):**
  - [ ] *Control Plane:* Redis-cluster (P2P-трекер, хранилище метаданных о чанках).
  - [ ] *Local Data Plane:* `storage-mounter` (FUSE-клиент) перехватывает запросы `vllm` и общается со `storage-agent` по быстрому UDS (Unix Domain Socket).
  - [ ] *P2P Network:* `storage-agent` на нодах общаются по TCP, находя нужные чанки через Redis.
  - [ ] *Fallback:* Скачивание недостающих чанков из HuggingFace и регистрация в Redis.

## 7. Стек технологий
- [ ] Язык разработки: Go (Golang).
- [ ] Инфраструктура: Kubernetes, Redis Cluster.
- [ ] Системные технологии: FUSE (Filesystem in Userspace), UDS, TCP.
- [ ] ML-инструменты: vLLM, HuggingFace.
