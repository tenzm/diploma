# Список определений и терминов (Definitions TODO)

Ниже представлен список терминов и определений, которые необходимо раскрыть во введении или глоссарии дипломной работы. Термины разделены по смысловым категориям.

## 1. Машинное обучение и Инференс
1. **Инференс (Inference)** — процесс использования обученной модели машинного обучения для получения предсказаний или генерации текста на основе новых входных данных.
2. **Холодный старт (Cold Start)** — задержка инициализации приложения (или контейнера с ML-моделью) при первом обращении к нему, вызванная необходимостью выделения ресурсов, скачивания образа, распаковки и загрузки весов модели в память.
3. **LLM (Large Language Model)** — большая языковая модель, нейронная сеть с миллиардами параметров (например, GPT, Llama, Falcon), требующая значительных объемов оперативной и видеопамяти для работы.
4. **Веса модели (Model Weights)** — числовые параметры нейронной сети, полученные в процессе обучения, определяющие логику работы модели. В современных LLM размер весов достигает сотен гигабайт.
5. **Чанк (Chunk)** — фрагмент файла фиксированного или переменного размера. В контексте работы — часть большого файла весов модели, используемая для гранулярного кэширования и передачи по сети.
6. **vLLM** — высокопроизводительный движок для инференса больших языковых моделей, оптимизированный для работы с памятью (PagedAttention).

## 2. Инфраструктура и Serverless
7. **Serverless (Бессерверные вычисления)** — модель выполнения облачных вычислений, при которой облачный провайдер (или оркестратор, такой как Knative) динамически управляет выделением вычислительных ресурсов, скрывая инфраструктуру от разработчика.
8. **Kubernetes (K8s)** — открытая платформа для автоматизации развертывания, масштабирования и управления контейнеризированными приложениями.
9. **Pod (Под)** — минимальная развертываемая вычислительная единица в Kubernetes, представляющая собой один или несколько контейнеров с общим хранилищем и сетью.
10. **DaemonSet** — контроллер в Kubernetes, который гарантирует, что копия определенного пода запущена на всех (или на некоторых выбранных) узлах кластера.
11. **Scale-to-zero** — механизм автомасштабирования в Serverless-архитектурах, при котором количество работающих экземпляров сервиса уменьшается до нуля при отсутствии входящего трафика для экономии ресурсов.
12. **SLA (Service Level Agreement)** — соглашение об уровне обслуживания, фиксирующее допустимые задержки (latency), доступность (availability) и пропускную способность сервиса.

## 3. Распределенные системы и P2P
13. **P2P-сеть (Peer-to-Peer)** — одноранговая децентрализованная компьютерная сеть, в которой все узлы (пиры) обладают равными правами и могут выступать как в роли клиента, так и в роли сервера.
14. **Control Plane (Плоскость управления)** — часть сетевой архитектуры, отвечающая за управление маршрутизацией, хранение метаданных и координацию узлов (в проекте реализуется через Redis).
15. **Data Plane (Плоскость данных)** — часть сетевой архитектуры, отвечающая за непосредственную передачу пользовательских данных (в проекте — передача чанков между `storage-agent`).
16. **Gossip Protocol (Эпидемический протокол)** — децентрализованный алгоритм распространения информации в P2P-сетях, при котором каждый узел периодически обменивается данными со случайно выбранными соседями.
17. **Leader Election (Выборы лидера)** — процесс в распределенных системах по определению единственного узла (лидера), который будет выполнять эксклюзивную задачу (например, скачивание файла из интернета).
18. **Единая точка отказа (SPOF - Single Point of Failure)** — компонент системы, отказ которого приводит к неработоспособности всей системы в целом (например, централизованный NFS-сервер).

## 4. Системное программирование и Файловые системы
19. **FUSE (Filesystem in Userspace)** — механизм в операционных системах семейства UNIX, позволяющий создавать виртуальные файловые системы в пространстве пользователя без модификации кода ядра.
20. **UDS (Unix Domain Socket)** — механизм межпроцессного взаимодействия (IPC) для обмена данными между процессами, выполняющимися на одной и той же операционной системе, обеспечивающий высокую скорость за счет отсутствия сетевого стека (TCP/IP).
21. **NFS (Network File System)** — протокол сетевого доступа к файловым системам, позволяющий монтировать удаленные директории по сети.
22. **File Locking (flock)** — механизм операционной системы, позволяющий процессу заблокировать файл для предотвращения одновременного доступа к нему со стороны других процессов.
23. **Cache Miss (Промах кэша)** — ситуация, при которой запрашиваемые данные отсутствуют в кэше, что требует их загрузки из более медленного источника (например, соседнего узла или внешнего репозитория).
24. **Fallback (Резервный механизм)** — запасной сценарий работы системы, который активируется в случае сбоя или отсутствия данных в основных источниках (например, скачивание напрямую с HuggingFace при отсутствии чанка в кластере).
